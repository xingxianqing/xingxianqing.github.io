iOS使用Jenkins+fir搭建持续集成环境

  • Jenkins
  • iOS

一、安装jenkins

$ brew install jenkins

如果没有安装java环境的话会出现以下错误，需要先安装jenkins需要安装java环境， JDK下载地址

安装完成后，启动jenkins，命令：

$ jenkins

在浏览器中输入以下地址访问：

http://localhost:8080/

访问成功后，找到系统管理->插件管理，然后添加一下4个插件：

  • Xcode integration

  • Git plugin

  • Post-Build Script Plug-in

  • GitHub plugin

安装完成后，重启jenkins!

二、新建项目

1.点击左侧工具栏的新建->填入项目名称-> 选择 构建一个自由风格的软件项目 ->点击 OK按钮 ->进入到配置页面

2.填入描述信息，找到源码管理这一项，我的在GitLab上，所以我这里选择Git，填入 Repository URL

3.可以通过Poll SCM 选项设置Jenkins在什么时候自动创建(比如：周一到周五，每天早上10点自动构建）,可以点击输入框后边的问号看说明。

4.找到构建选项中增加构建步骤，选择Xcode

  • Target ：填写项目中Target的名字
  • Clean before build 设置为： YES
  • Configuration 设置为：Release
  • .ipa filename pattern 起一个.ipa文件的名字，不需要写后缀
  • Output directory 是.ipa的输出路径，我填的是：${WORKSPACE}/build/

5.在下边的Code signing & OS X keychain options选项中，设置证书信息，之前在项目中就已经设置过证书了，在这里只要填入路径就可以了

步骤：

  • 勾选Unlock KeyChain？选项，

  • 在KeyChain Path的输入框中填入：

  • Keychain password的输入框中填入：

6.如果使用了cocoapods，需要设置一下Advanced Xcode build options选项。

  • Schema 项目的schema名称
  • SDK 设置为：iphoneos
  • Xcode Workspace File 填写workspace的文件名字，不需要后缀
  • Build output directory 设置为：${WORKSPACE}/build/

注意:在下一步之前，先安装fir.im 的工具fir-cli，命令：

$ sudo gem install fir-cli –no-ri –no-rdoc

7.点击增加构建后操作->选择Execute a set of scripts->点击Add Build Step->选择Execute shell，输入框中填入：

fir的APIToken需要登录fir获取

设置完成之后可以点击保存了！

8.保存之后点击立即构建，就可以看见开始构建了，蓝色是构建成功，红色为失败

三、邮件配置

1.在首页找到系统管理选项并进入->选择系统设置选项->找到系统管理员邮件地址，并填入你的邮箱。

2.找到邮件通知选项进行配置，（如果你的邮箱不支持SMTP，要先去邮箱账户里设置一下）

  • SMTP服务器：根据自己的邮箱配置，我用的是QQ企业邮箱，填的是 smtp.exmail.qq.com
  • 勾选使用SMTP认证
  • 用户名： 邮箱地址
  • 密码： 密码 ，如果邮箱开启了“微信动态码，这里要填授权码”
  • 勾选使用SSL协议认证
  • 端口：根据自己邮箱SMTP服务器来配置
  • 字符集： UTF-8
  • 勾选通过发送测试邮件测试配置选项
  • Test e-mail recipient：邮箱地址
  • 点击Test configuration按钮进行测试，输入框下方出现Email was successfully sent,表示测试成功，你会收到一封内容为This is test email #9 sent from Jenkins的邮件,然后保存当前设置即可！

3.进入你的项目->在左侧找到配置选项并进入配置页面，找到增加构建后步骤选项并点击->选择E-mail Notification，在Recipients 填入邮箱地址，如果要发给多个人，以空格分隔，这样在构建失败的时候都会发送邮件给这个接收者。
